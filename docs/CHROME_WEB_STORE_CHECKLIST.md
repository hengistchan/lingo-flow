# Chrome Web Store Readiness Checklist

Use this checklist before submitting LingoFlow to the Chrome Web Store. Do not submit until all items are complete.

## Extension Identity

- [ ] **Name:** LingoFlow
- [ ] **Short description:** AI-powered bilingual web reading browser extension
- [ ] **Detailed description:** See store listing draft below
- [ ] **Category:** Productivity
- [ ] **Language:** English (primary), Simplified Chinese (UI bilingual support)

## Icons and Images

- [ ] **128x128 icon:** Required for Chrome Web Store listing — `apps/extension/assets/lingoflow-icon.svg` (convert to PNG)
- [ ] **16x16 icon:** Used in toolbar (generated by WXT from source SVG)
- [ ] **32x32 icon:** Used in management page (generated by WXT from source SVG)
- [ ] **48x48 icon:** Used in extensions page (generated by WXT from source SVG)
- [ ] **Screenshots (1280x800 or 640x400):** At least 1, recommended 3-5
  - [ ] Popup showing translation in progress
  - [ ] Options page with provider configuration
  - [ ] Translated page showing bilingual inline rendering
- [ ] **Promotional tile (440x280):** Optional but recommended
- [ ] **Marquee (1400x560):** Optional, for featured placement

## Privacy Disclosure

- [ ] Does the extension collect user data? **No**
- [ ] Does the extension transmit user data? **Yes** — page text is sent to the user-configured translation provider
- [ ] Does the extension use analytics? **No**
- [ ] Does the extension use remote code? **No** — all code is bundled in the extension
- [ ] Privacy policy URL: Link to `docs/PRIVACY.md` in the repository

## Permissions Justification

| Permission | Justification |
|---|---|
| `activeTab` | Access the current tab when the user clicks Translate |
| `scripting` | Inject content script to collect and render translations |
| `storage` | Store settings, API keys, and translation cache locally |

## Host Permission Explanation

| Host Permission | Reason |
|---|---|
| `https://api.cognitive.microsofttranslator.com/*` | Azure Translator provider |
| `https://api.openai.com/*` | OpenAI and OpenAI-compatible providers |
| `https://translate.googleapis.com/*` | Google Translate Free provider |

**Optional host permissions** (`https://*/*`, `http://*/*`): Not granted at install. Requested only when user adds a custom provider endpoint.

## Single-Purpose Policy

LingoFlow has a single purpose: translating web page text into the user's target language and displaying the translation inline. All permissions and functionality serve this purpose.

## Data Usage Disclosure

- [ ] The extension does not sell user data to third parties.
- [ ] The extension does not use user data for purposes unrelated to its core functionality.
- [ ] The extension does not use user data for creditworthiness or lending purposes.
- [ ] Page text is transmitted to the user's chosen translation provider solely for the purpose of translation.
- [ ] API keys are stored locally and never transmitted to LingoFlow.

## Testing Checklist

- [ ] Extension installs without errors on a clean Chrome profile.
- [ ] Popup opens and shows "Ready to translate".
- [ ] Google Translate Free default works without configuration.
- [ ] Azure Translator works with valid API key.
- [ ] OpenAI-compatible provider works with valid API key and endpoint.
- [ ] Custom provider endpoint works after granting optional host permission.
- [ ] "Test connection" button succeeds/fails correctly.
- [ ] Translate a simple article page — translations appear inline.
- [ ] Clear translation — original DOM is restored.
- [ ] Display modes (original, dual, translation) work correctly.
- [ ] Dark mode renders correctly.
- [ ] Options page loads and saves settings.
- [ ] Extension does not request `<all_urls>` at install.
- [ ] No console errors during normal operation.
- [ ] Extension works on GitHub, Wikipedia, and generic article pages.

## Release ZIP

- [ ] Release ZIP generated with `pnpm package`.
- [ ] ZIP location: `apps/extension/output/lingoflowextension-<version>-chrome.zip`.
- [ ] ZIP contains only extension files (no source maps, node_modules, test fixtures, or env files).
- [ ] Manifest version matches `package.json` version.
- [ ] No secrets or API keys in any file inside the ZIP.

## Rollback Notes

- The Chrome Web Store allows uploading new versions. If a release has critical issues, upload the previous known-good ZIP as a new version.
- Keep the previous release ZIP archived locally.
- Document the last known-good version in the changelog.
